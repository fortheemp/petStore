# PetStore 宠物商城项目汇报

## 一、项目简介

PetStore 是一个全平台宠物商品商城系统，采用前后端分离架构，包含 PC Web 端、uni-app 手机端和管理后台三个客户端，共用同一套 REST API 后端。系统涵盖商品浏览、购物车、订单流转、宠物视频社区、AI 智能助手等功能。

## 二、技术架构

| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端（PC） | Vue 3 + Element Plus + Vite | 单页应用，组件化开发 |
| 手机端 | uni-app（Vue 3） | 一套代码编译为 Android APK |
| 后端 | Spring Boot 2.7 + Spring Data JPA | RESTful API 服务 |
| 数据库 | SQLite | 轻量级文件数据库，开启外键约束 |
| AI 助手 | DeepSeek API（OpenAI 兼容格式） | 大语言模型对话 + 商品推荐 |
| 部署 | Nginx 静态资源 + Spring Boot 内嵌 Tomcat | 8080 端口 |

## 三、数据库设计

共 9 张表，使用 SQLite + 外键约束：

### 核心表结构

| 表名 | 主要字段 | 说明 |
|------|---------|------|
| **users** | id, username(UNIQUE), password, nickname, avatar, role, phone | 角色：admin/member/tourist |
| **shops** | id, name, address, longitude, latitude, image, rating | 支持经纬度的门店模型 |
| **products** | id, shop_id(FK), name, type, stock, price, image, subcategory | 类型：pet(宠物)/accessory(用品) |
| **videos** | id, title, url, cover, category, product_id | 视频关联推荐商品 |
| **orders** | id, user_id(FK), address_snapshot, total_amount, status, refund_reason | 整数状态码流转 |
| **order_items** | id, order_id(FK), product_name, product_image, price, quantity | 冗余存储商品快照 |
| **cart_items** | id, user_id(FK), product_id(FK), quantity | 用户购物车 |
| **addresses** | id, user_id(FK), receiver_name, phone, province/city/district/detail | 支持默认地址 |
| **reviews** | id, order_id(FK), user_id(FK), content, rating | 按订单维度评价 |

### 订单状态流转

```
状态码：0(待付款) → 1(已付款) → 2(已发货) → 3(已完成)
                                    ↓
                              退款申请 → 审核通过(退款完成) / 审核拒绝(恢复原状态)
```

通过 `previous_status` 字段记录状态变更前的状态，支持审核拒绝时回退。

## 四、后端核心代码

### 4.1 分层架构

```
Controller（接收请求）→ Service（业务逻辑）→ Repository（数据访问）→ Entity（数据模型）
```

- **Controller 层**：11 个控制器，使用 `@RestController`，返回统一 `Result<T>` 格式
- **Service 层**：10 个服务类，使用 `@Service` + `@Transactional`
- **Repository 层**：9 个接口，继承 `JpaRepository`，自动生成 CRUD
- **Entity 层**：9 个实体类，使用 JPA 注解映射数据库表

### 4.2 统一响应格式

```java
// common/Result.java
public class Result<T> {
    private int code;    // 200=成功
    private String msg;
    private T data;
}
```

所有 API 返回 `{code, msg, data}` 格式，前端 axios 拦截器自动解包，直接返回 `data`。

### 4.3 主要 API 接口

**商品模块：**
- `GET /api/products` — 商品列表（支持 shopId、type、keyword、subcategory 筛选）
- `GET /api/products/{id}` — 商品详情
- `GET /api/products/{id}/reviews` — 商品评价

**订单模块：**
- `POST /api/orders` — 创建订单（从购物车结算，校验库存）
- `POST /api/orders/{id}/pay` — 模拟支付
- `POST /api/orders/{id}/cancel` — 取消订单（含原因）
- `POST /api/orders/{id}/confirm` — 确认收货
- `POST /api/orders/{id}/refund` — 申请退款
- `POST /api/orders/{id}/review` — 提交评价

**购物车模块：**
- `GET /api/cart` — 购物车列表
- `POST /api/cart` — 添加商品（校验库存）
- `PUT /api/cart/{id}` — 修改数量
- `DELETE /api/cart/{id}` — 删除商品

**AI 模块：**
- `POST /api/ai/chat` — AI 对话（传入 question，返回 AI 回复 + 推荐商品）

**管理后台：**
- `/api/admin/shops`、`/api/admin/products`、`/api/admin/videos`、`/api/admin/users` — 各模块 CRUD
- `/api/admin/orders/{id}/ship` — 发货
- `/api/admin/orders/{id}/approve-refund` — 审核退款（含拒绝理由）

## 六、关键业务流程

### 购物下单流程

```
1. 用户浏览商品 → 加入购物车（校验库存）
2. 购物车页面 → 调整数量 → 点击结算
3. 结算页 → 选择收货地址 → 二次校验库存
4. 提交订单 → 扣减库存 → 生成订单（status=0 待付款）
5. 点击支付 → 模拟付款（status=1 已付款）
6. 管理员发货 → status=2 已发货
7. 用户确认收货 → status=3 已完成 → 可评价
```

### 退款流程

```
1. 用户申请退款（填写原因）→ 记录 previous_status
2. 管理员审核 → 通过：恢复库存，状态更新为退款完成
                → 拒绝：填写拒绝理由，回退到 previous_status
```

### AI 推荐流程

```
1. 用户输入问题（如"推荐一只适合新手养的猫"）
2. 后端将商品列表注入系统提示词 → 调用 DeepSeek API
3. AI 回复 + 输出【推荐:4,6】格式的推荐标记
4. 后端返回纯文本 → 前端解析推荐 ID
5. 前端查找对应商品 → 展示推荐卡片（含商品图片、名称、价格）
```

## 八、总结

本项目完成了一个功能完整的全平台宠物商城系统：

1. **前后端分离架构**：Vue 3 + Spring Boot + uni-app 三端共用同一后端
2. **完整业务闭环**：商品浏览 → 购物车 → 下单 → 支付 → 发货 → 收货 → 评价，含退款流程
3. **AI 智能推荐**：对接 DeepSeek 大模型，基于商品上下文实现智能问答与推荐
4. **多端适配**：PC Web 端、手机端 Android APK，响应式设计
5. **安全合规**：通过 OWASP + CWE 安全扫描，0 漏洞
