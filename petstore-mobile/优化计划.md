# 宠物商城移动端 - 优化计划（基于 PC 端对比）

## 一、PC 端与移动端功能对比

### 页面对比

| 功能模块 | PC 端 | 移动端 | 状态 |
|----------|-------|--------|------|
| 首页 | HeroBanner + 分类卡片 + 热门商品 + 附近店铺 + 服务卡片 | HeroBanner + 分类网格 + 热门商品 + 推荐商品 | 缺少附近店铺模块 |
| 商品列表 | 侧边栏筛选 + 分页 + 分类图标行 + 面包屑 | 搜索 + 标签筛选 + 分类chips + 无限滚动 | 功能基本对齐 |
| 商品详情 | 图片画廊 + 视频播放 + 规格选择 + 收藏 + 关联推荐 | 图片占位 + 规格选择 + Tab切换 | 缺少：视频、收藏、关联推荐 |
| 购物车 | Pinia store + 后端 API 同步 + 邮费计算 | reactive store + localStorage | 功能基本对齐，邮费计算缺失 |
| 订单管理 | 5 状态 Tab + 退款/评价/取消/确认收货 | 5 状态 Tab + 取消/确认收货/评价 | 缺少：退款功能 |
| 地址管理 | 省市区三级联动 + CRUD + 设为默认 | 省市区选择 + CRUD（内存数据，未持久化） | 地址数据未持久化 |
| 附近店铺 | **高德地图** + POI 搜索 + 标记点 + 距离筛选 + 导航 | 列表展示 + 搜索 | 缺少：地图、导航 |
| 视频 | 分类标签 + 视频播放器 + 关联商品 | 视频列表（点击仅 toast） | 缺少：真实播放器 |
| 收藏夹 | 本地收藏列表页 | 无此功能 | 完全缺失 |
| AI 助手 | 浮动窗口 + 打字机效果 + 商品推荐 | 全屏聊天页 + 打字机效果 + 商品推荐 | 功能对齐，交互方式不同 |
| 后台管理 | 完整的 Admin 后台（仪表盘/商品/订单/用户/视频管理） | 无 | 移动端不需要，合理 |
| 登录/注册 | 表单验证 + 记住密码 + 跳转 | 表单验证 + 记住密码 + 短信验证 + 密码强度 | 移动端功能更丰富 |

### 技术架构对比

| 维度 | PC 端 | 移动端 | 差距 |
|------|-------|--------|------|
| 状态管理 | **Pinia**（5 个 store：user/cart/order/address/admin） | **reactive()**（2 个 store：user/cart） | 缺少 order、address store |
| API 层 | 8 个模块，全部调用真实后端 | 6 个模块，**全部使用 mock 数据** | 需全面对接 |
| HTTP 客户端 | Axios + 拦截器 + 信封解包 | uni.request + Bearer Token | 功能基本对齐 |
| Token 读取 | `localStorage.getItem('petstore_token')` | `uni.getStorageSync('token')` | **Key 不一致，需修复** |
| 导航守卫 | vue-router beforeEach 守卫 | 无 | 需添加 |
| 骨架屏 | 多个页面有 | 有部分 | 基本对齐 |
| 设计系统 | CSS 变量（variables.css） | SCSS 变量（uni.scss） | 风格一致，实现方式不同 |

## 二、当前移动端存在的 Bug

| # | 问题 | 位置 | 严重程度 |
|---|------|------|----------|
| 1 | Token 存储 Key 不一致：`request.js` 读 `'token'`，`stores/user.js` 写 `'petstore_token'` | `api/request.js` + `stores/user.js` | **高** — HTTP 请求永远无法带上认证 Token |
| 2 | 登录方法签名不匹配：页面传 `loginForm` 对象，store 期望 `(token, user)` 两个参数 | `pages/auth/login.vue` + `stores/user.js` | **高** — 登录后 token 和 user 均为错误值 |
| 3 | 用户数据存储 Key 不一致：`api/user.js` 写 `'petstore_user_info'`，`stores/user.js` 读 `'petstore_user'` | `api/user.js` + `stores/user.js` | **高** — 用户信息读取为空 |
| 4 | 店铺详情页不存在：`shop/nearby.vue` 导航到 `/pages/shop/detail?id=X`，但该页面未创建 | `pages/shop/nearby.vue` | **中** — 点击会报错 |
| 5 | 结算页地址选择未连通：地址页 emit `selectAddress` 事件，但结算页未监听 | `pages/order/checkout.vue` + `pages/user/address.vue` | **中** — 无法选中地址 |
| 6 | 地址数据仅存内存：添加/编辑/删除后刷新即丢失 | `pages/user/address.vue` | **中** |
| 7 | `custom-tabbar` 组件已定义但未使用 | `components/custom-tabbar/` | **低** |

## 三、优化计划

### 阶段 1：修复 Bug + 统一规范（优先）

> 目标：让现有功能稳定可用，消除不一致

#### 1.1 统一 Token 存储 Key
- 修改 `api/request.js`：将 `uni.getStorageSync('token')` 改为 `uni.getStorageSync('petstore_token')`
- 确保所有涉及 Token 读写的地方使用同一个 Key

#### 1.2 修复登录/注册方法签名
- 修改 `stores/user.js` 的 `login()` 方法，使其能接收页面传入的表单对象
- 或修改 `pages/auth/login.vue` 的调用方式，适配 store 的参数格式
- 同步修复注册页面

#### 1.3 统一用户数据存储 Key
- 统一 `api/user.js` 和 `stores/user.js` 中的用户数据 Key 为 `'petstore_user'`

#### 1.4 地址数据持久化
- 将 `pages/user/address.vue` 中的地址数据存入 localStorage
- 或在 `stores/` 下新建 `address.js` store 管理地址状态

#### 1.5 结算页地址选择连通
- 在 `pages/order/checkout.vue` 中监听 `uni.$on('selectAddress')` 事件
- 或通过 `uni.$once` 一次性监听避免重复

---

### 阶段 2：状态管理升级

> 目标：对齐 PC 端的 Pinia 架构，为后端对接做准备

#### 2.1 引入 Pinia
- uni-app 官方支持 Pinia，直接引入即可
- 创建 `stores/index.js` 初始化 Pinia 实例
- 在 `main.js` 中 `app.use(createPinia())`

#### 2.2 新增 Order Store
- 参考 PC 端 `stores/order.js`
- 管理订单列表、状态筛选、订单操作（取消/确认收货/评价）
- 当前对接 localStorage，后端就绪后切换为 API 调用

#### 2.3 新增 Address Store
- 参考 PC 端 `stores/address.js`
- CRUD 操作 + 设为默认 + 获取默认地址
- 替换 `user/address.vue` 中的内存数据

#### 2.4 迁移现有 Store
- 将 `stores/user.js` 和 `stores/cart.js` 从 reactive() 迁移到 Pinia defineStore
- 保持现有功能不变，仅改变实现方式

---

### 阶段 3：功能补全（对齐 PC 端）

> 目标：补齐移动端缺失的核心功能

#### 3.1 店铺详情页（新建）
- 创建 `pages/shop/detail.vue`（或 `pages/shop/detail/detail.vue`）
- 展示店铺信息：名称、地址、评分、简介
- 展示该店铺下的商品列表
- 参考 PC 端 `shop.js` API

#### 3.2 商品收藏功能
- 新建 `pages/user/favorites.vue` 收藏列表页
- 商品详情页添加收藏按钮（心形图标，localStorage 持久化）
- 个人中心菜单增加"我的收藏"入口

#### 3.3 商品关联推荐
- 商品详情页底部增加"猜你喜欢"或"相关商品"推荐
- 参考 PC 端 `ProductDetail.vue` 的 relatedProducts 逻辑

#### 3.4 视频播放器
- `pages/video/list.vue` 点击视频卡片时跳转到视频播放页
- 或在当前页面实现内嵌播放器（使用 uni-app 的 `<video>` 组件）
- 支持分类筛选（全部/狗狗/猫咪/训练/搞笑）

#### 3.5 邮费计算
- 购物车和结算页加入邮费逻辑
- 参考 PC 端：满 199 免邮，否则 10 元运费
- 在 cart store 中增加 `shipping` 和 `total` 计算

#### 3.6 订单退款功能
- 订单详情页增加"申请退款"按钮
- 订单列表增加"退款"状态 Tab
- 参考 PC 端 `order.js` 中的 `refundOrder` API

#### 3.7 导航守卫（uni-app 路由拦截）
- 使用 `uni.addInterceptor` 或页面 `onLoad` 中做登录态检查
- 保护需要登录的页面：订单列表、结算页、地址管理、个人中心

#### 3.8 附近店铺地图（可选）
- uni-app 中集成高德地图需使用 `uni.getLocation` + `map` 组件
- 或简化为：点击店铺调用手机地图 App 进行导航（使用 `uni.openLocation`）
- PC 端使用了 `@amap/amap-jsapi-loader`，移动端需改用 uni-app 方案

---

### 阶段 4：代码清理 + 架构优化

> 目标：提升代码质量和可维护性

#### 4.1 清理 Mock 数据
- 移除各页面中的内联硬编码数据（product/list、product/detail、order/list、order/detail、shop/nearby、index）
- 所有数据统一从 `api/` 层或 `stores/` 获取
- 等后端就绪后，将 `api/*.js` 中的 mock 调用替换为 `request.js` 的真实 HTTP 请求

#### 4.2 移除未使用的 API 模块
- 当前 `api/cart.js` 和 `api/order.js` 未被任何页面使用
- 后端对接后可直接替换为真实调用
- 当前可暂时保留，标注为待对接状态

#### 4.3 提取公共组件
- 首页和分类页的自定义导航栏（蓝色渐变头部）提取为公共组件 `custom-header`
- 商品卡片提取为可复用组件（类似 PC 端的 `ProductCard.vue`）
- 空状态组件统一（已有 `.empty-state` 样式，可组件化）

#### 4.4 清理 Git 追踪
- 确保 `unpackage/` 目录在 `.gitignore` 中（当前已有，但可能已被追踪）
- 执行 `git rm -r --cached unpackage/` 清理

---

### 阶段 5：后端对接（等后端就绪）

> 目标：将所有 mock 替换为真实 API

#### 5.1 API 模块替换顺序（建议）
1. `api/user.js` → 登录/注册/个人信息（最基础，最先对接）
2. `api/product.js` → 商品列表/详情/搜索
3. `api/cart.js` → 购物车 CRUD
4. `api/order.js` → 订单创建/列表/详情/操作
5. `api/address.js` → 地址 CRUD
6. `api/shop.js` → 店铺列表/详情

#### 5.2 对接要点
- 确保 `request.js` 的 base URL 与后端一致（`http://localhost:8080/api`）
- 统一请求/响应格式，参考 PC 端的信封解包逻辑（`{code, data}` 格式）
- 处理后端返回的错误码和错误消息
- Token 传递方式与后端对齐（Bearer Token）
