

# 一、需求规格说明书

## 1.1 项目背景与目标
- **项目名称**：宠物商店（PetStore）
- **开发模式**：前后端分离
- **目标**：构建一个集宠物及周边商品浏览、购买、订单管理、地图导航、AI问答、视频展示于一体的综合电商平台，满足课程实训要求。

## 1.2 用户角色
| 角色 | 职责描述 |
|------|----------|
| 游客 | 浏览商店、商品、视频，不能购买 |
| 会员 | 注册登录后，可购买商品、管理订单、评价 |
| 管理员 | 登录后可在前端管理界面动态添加/编辑商店、商品（上传图片），管理视频、会员、订单（含退单审核） |

## 1.3 基础功能需求（必须实现）

### 1.3.1 前端功能

#### 1. 注册与登录
- 支持用户名+密码注册/登录，密码明文存储
- 登录后获取会员身份，可进行购买操作
- 管理员账号预先内置或由管理员注册，登录后跳转至管理界面

#### 2. 个人中心
- 查看/修改个人信息（昵称、头像等）
- 管理收货地址（支持多个地址，可设置默认地址）
- 字段校验（前端+后端）

#### 3. 附近宠物商店浏览
- 以地图形式展示本地数据库中的商店位置（商店图片在地图标记或详情中显示）
- 支持商店详细地址查看（含商店图片）
- 支持从当前位置到商店的路线导航（调用高德/百度地图API）
- 地图搜索结果显示本地商店信息

#### 4. 商品浏览与购买
- 游客可浏览所有商店和商品（商品列表与详情中展示商品图片）
- 会员可将商品加入购物车
- 商品分为两类：**宠物**（唯一库存）和**宠物周边**（库存不限）
- 商品属于某个商店，不同商店可有相同商品
- 生成订单时选择数量、规格，完成下单

#### 5. 管理员前端管理功能（基础）
- **商店管理**：管理员登录后可见“商店管理”界面，可动态添加商店（填写名称、地址、经纬度，本地上传商店图片），编辑或删除商店
- **商品管理**：管理员可动态添加商品（填写名称、类型、库存、价格，本地上传商品图片，关联商店），编辑或删除商品
- **视频管理**：上传视频，关联商品
- **会员管理**：查看注册用户列表
- **订单管理**：查看所有订单，处理退单申请（审核通过/不通过，直接退单），修改订单状态

#### 6. 订单管理（完整状态流转）
- **订单状态**：
    - 0：已下单（待支付）
    - 1：已支付（待发货）
    - 2：已发货（待收货）
    - 3：已收货（待评价）
    - 4：已评价（结束）
    - -1：取消订单（仅状态0、1可取消，记录取消原因：用户取消/超时自动取消）
    - -2：申请退单（状态2、3可申请，需录入理由）
    - -3：管理员审核通过，退单成功（结束）
    - -4：管理员直接退单（已收货可操作，需录入理由）
- 若退单审核不通过，订单状态恢复为原状态，可继续申请
- 订单完成后可评价

#### 7. 宠物视频
- 视频列表页，支持播放
- 视频可链接到对应商品
- 商品详情中嵌入视频介绍
- 视频由管理员后台上传（基础功能）

#### 8. AI智能问答助手
- 调用大模型接口（如通义千问）
- 实现宠物喂养知识问答、商品推荐、客服咨询

### 1.3.2 非功能需求
- **安全性**：Java代码经SAST扫描满足GB/T 34944，无高危漏洞；系统经AppScan扫描无中高危漏洞（因无加密与鉴权，需注意其他注入等风险）
- **字段校验**：所有输入均需前后端双重校验
- **项目管理**：使用禅道/飞书进行任务管理，Gitee进行版本控制，提供会议记录
- **文档**：交付需求文档、设计文档、演示PPT

# 二、系统设计文档

## 2.1 系统架构

```
前端（Vue/Uni-app） → SpringBoot 后端服务（单体或微服务）
                          ├── 用户服务
                          ├── 商店服务
                          ├── 商品服务
                          ├── 订单服务
                          ├── 视频服务
                          ├── AI问答服务
                          └── 文件上传服务
数据库：sqlite（业务数据） + Redis（缓存/会话可选） + MongoDB（可选，附近搜索）
文件存储：放在项目根目录
第三方：高德/百度地图API、大模型API
```

## 2.2 技术选型
| 层次 | 技术 |
|------|------|
| 前端 | Vue 3 + Element UI / Vant，或 Uni-app（跨端） |
| 后端 | SpringBoot 
| 数据库 | sqlite
| 项目管理 | 禅道/飞书，Gitee |
| 代码扫描 | SAST工具（厦理工实验室），AppScan |
| IDE | VSCode（前端），IDEA（后端） |

## 2.3 模块划分与接口设计（概要）

### 2.3.1 用户模块
- `POST /api/user/register` 注册（密码明文）
- `POST /api/user/login` 登录（验证账号密码，返回用户基本信息及角色）
- `GET/PUT /api/user/profile` 个人信息（无需Token，前端根据登录状态发送请求）
- `POST/GET/PUT/DELETE /api/user/address` 收货地址管理

### 2.3.2 商店模块
- `GET /api/shops` 获取附近商店（传入经纬度，返回距离排序，含图片）
- `GET /api/shops/{id}` 商店详情（含图片）
- `POST /api/admin/shops` 管理员添加商店（接收表单数据，含图片文件上传）
- `PUT /api/admin/shops/{id}` 管理员编辑商店（可更新图片）
- `DELETE /api/admin/shops/{id}` 管理员删除商店

### 2.3.3 商品模块
- `GET /api/products?shopId=&type=` 按商店/类型查询商品（含图片）
- `GET /api/products/{id}` 商品详情（含图片、视频）
- `POST /api/admin/products` 管理员添加商品（接收表单数据，含图片上传，关联商店）
- `PUT /api/admin/products/{id}` 管理员编辑商品
- `DELETE /api/admin/products/{id}` 管理员删除商品

### 2.3.4 订单模块
- `POST /api/orders` 创建订单（从购物车生成）
- `PUT /api/orders/{id}/pay` 支付（模拟）
- `PUT /api/orders/{id}/cancel` 取消订单（需原因）
- `PUT /api/orders/{id}/refund-apply` 申请退单（需理由）
- `PUT /api/admin/orders/{id}/refund-approve` 审核退单
- `PUT /api/admin/orders/{id}/refund-direct` 直接退单
- `PUT /api/orders/{id}/confirm-receipt` 确认收货
- `POST /api/orders/{id}/review` 评价

### 2.3.5 视频模块
- `GET /api/videos` 视频列表
- `GET /api/videos/{id}` 视频详情
- `POST /api/admin/videos` 管理员上传视频（文件上传）
- `DELETE /api/admin/videos/{id}` 删除视频

### 2.3.6 AI问答模块
- `POST /api/ai/chat` 发送问题，返回大模型回答

### 2.3.7 文件上传接口
- `POST /api/upload/image` 上传图片（商店/商品图片），返回图片访问URL

## 2.5 安全设计
- 密码明文存储，不使用加密（简化实现）
- 接口不采用JWT等鉴权机制，管理员功能通过前端路由控制及后端简单角色判断（例如从请求参数或Session中获取用户角色，但非强制）
- 输入校验（前端正则+后端@Valid）防止SQL注入、XSS等
- SQL注入防护（MyBatis参数化查询）
- 文件上传限制类型与大小，防止恶意文件
- 代码扫描工具集成，确保无高危漏洞

## 2.6 管理员动态添加商店/商品流程
1. 管理员登录后，前端显示“管理后台”入口
2. 进入商店管理页面，点击“添加商店”，填写表单（名称、地址、经纬度），点击上传商店图片，调用 `/api/upload/image` 上传图片，获得图片URL后随表单提交到 `/api/admin/shops`
3. 商品管理类似：添加商品时选择所属商店，上传商品图片，提交至 `/api/admin/products`
4. 后端接收 multipart/form-data 请求，保存图片文件至服务器指定目录，数据库存储图片路径
