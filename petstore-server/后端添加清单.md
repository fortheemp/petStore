# 后端添加清单 — 评价功能

> 以下均为前端当前需要但后端尚未支持的功能。
> 请在 `petstore-server` 中实现，实现后告知前端同事即可对接。

---

## 1. Review 表添加 `product_id` 字段

**现状：** `reviews` 表只有 `order_id`、`user_id`、`content`、`rating`，没有商品维度。
**问题：** 前端一个订单可能有多个商品，需要给每个商品单独评价。

```sql
ALTER TABLE reviews ADD COLUMN product_id INTEGER;
```

`Review.java` 实体同步添加：
```java
@Column(name = "product_id")
private Long productId;
```

同时建议在 `(order_id, product_id)` 上加唯一约束，防止重复评价同一商品。

---

## 2. 评价接口支持多商品评价（修改现有接口）

**现状：** `POST /api/orders/{id}/review` 接收 `{ userId, content, rating }`，只创建一条 Review。
**需要改为：** 接收一个评价数组，每个商品一条。

**请求体改为：**
```json
{
  "userId": 1,
  "reviews": [
    { "productId": 10, "rating": 5, "content": "很好" },
    { "productId": 11, "rating": 4, "content": "还行" }
  ]
}
```

`OrderController.review()` 和 `OrderService.review()` 需要遍历数组，为每个商品创建一条 Review 记录。

---

## 3. 新增：获取订单评价列表接口

**现状：** `ReviewService.getByOrderId()` 已存在，但没有 Controller 暴露。
**前端需要：** 订单详情页提交评价后展示已评价内容。

**建议新增：**
```
GET /api/orders/{id}/reviews
```

**响应：**
```json
{
  "code": 0,
  "message": "success",
  "data": [
    {
      "id": 1,
      "orderId": 5,
      "productId": 10,
      "userId": 1,
      "username": "张三",
      "content": "很好",
      "rating": 5,
      "createdAt": "2026-06-25T10:30:00"
    }
  ]
}
```

> 注意：需要 JOIN `users` 表获取 `username`，或者 Review 表冗余存储 `username`。

**问题：** 没有按 `productId` 查询评价的接口。

```
GET /api/products/{id}/reviews
```

```java
List<Review> findByProductId(Long productId);
```

## 5. 新增：商品平均评分 & 评价数

**现状：** `ProductCard.vue` 显示 `rating`（平均评分）和 `reviewCount`（评价数）。
**问题：** 后端 Product 表没有这两个字段，也没有计算接口。

**两种方案（选一）：**

**方案 A（推荐）：Product 表加字段，评价时自动更新**
```sql
ALTER TABLE products ADD COLUMN avg_rating REAL DEFAULT 0;
ALTER TABLE products ADD COLUMN review_count INTEGER DEFAULT 0;
```
每次新增评价时重新计算 `avg_rating` 和 `review_count` 并回写 Product 表。
优点：查询快，商品列表接口直接返回。

**方案 B：单独接口实时计算**
```
GET /api/products/{id}/rating
```
返回 `{ avgRating: 4.5, reviewCount: 12 }`。
优点：不改 Product 表；缺点：列表页需要额外请求或批量计算。

## 6. 商品列表/详情接口返回评分信息

**现状：** `GET /api/products` 和 `GET /api/products/{id}` 不返回评分。
**需要：** 响应中包含 `avgRating`（或 `rating`）和 `reviewCount` 字段，前端直接使用。

如果选方案 A，在 Product DTO 中加上这两个字段即可。
如果选方案 B，前端需要在 ProductList 和 ProductDetail 页面额外调用评分接口。

---

## 7. `GET /api/orders/{id}` 响应补充评价状态

**现状：** 订单详情接口返回的 order 对象没有 `reviewed` 字段。
**前端需要：** 判断是否已评价，决定显示"写评价"还是"已评价"。

**建议：** 在订单详情接口中返回：
```json
{
  "order": {
    "id": 5,
    "status": 4,
    "reviewed": true
    
  },
  "items": []
}
```

可以在后端通过查询 `reviews` 表判断该订单是否已有评价记录。

---

## 8. `GET /api/orders` 列表接口补充评价状态

**现状：** 订单列表接口返回的订单对象也没有 `reviewed` 字段。
**前端需要：** 订单列表中 status=3 的订单显示"写评价"按钮，status=4 且已评价的显示"已评价"。

在列表接口的每个 order 中也加上 `reviewed` 字段。

## 9. shops 表新增商店信息字段

**现状：** `shops` 表只有 `id`, `name`, `address`, `longitude`, `latitude`, `image`, `created_at`。
**问题：** 前端 NearbyShops 页面还需要以下字段，目前均为空。

```sql
ALTER TABLE shops ADD COLUMN status VARCHAR(10) DEFAULT 'open';
ALTER TABLE shops ADD COLUMN rating REAL DEFAULT 0;
ALTER TABLE shops ADD COLUMN review_count INTEGER DEFAULT 0;
ALTER TABLE shops ADD COLUMN phone VARCHAR(20);
ALTER TABLE shops ADD COLUMN business_hours VARCHAR(50);
```

`Shop.java` 实体同步添加：
```java
private String status;        // "open" 或 "closed"
private Double rating;        // 商店评分
@Column(name = "review_count")
private Integer reviewCount;  // 评价数量
private String phone;         // 联系电话
@Column(name = "business_hours")
private String businessHours; // 营业时间，如 "09:00-21:00"
```

> **评分同步：** `rating` 和 `review_count` 建议在新增评价时自动重新计算并回写 `shops` 表（类似商品评分方案）。
> 也可以从 `reviews` 表实时 JOIN 计算，但列表查询会较慢。

## 10. 新增：按区域查询商店接口（可选）

**现状：** `GET /api/shops` 返回全部商店，前端通过地址文本匹配区域名。
**问题：** 硬靠地址字符串 `LIKE` 匹配不够精确，建议后端支持区域筛选。

**建议新增参数：**
```
GET /api/shops?district=朝阳区
```

`ShopRepository` 需新增：
```java
List<Shop> findByAddressContaining(String district);
```

此需求优先级较低，前端当前已通过地址文本匹配实现，不实现也可用。

## 11. videos 表新增字段

**现状：** `videos` 表只有 `id`, `title`, `url`, `product_id`, `created_at`。
**问题：** 前端 VideoList 页面需要更多字段来展示视频信息。

```sql
ALTER TABLE videos ADD COLUMN duration VARCHAR(10);        -- 显示用，如 "03:25"
ALTER TABLE videos ADD COLUMN duration_seconds INTEGER;    -- 播放器进度条用
ALTER TABLE videos ADD COLUMN view_count INTEGER DEFAULT 0;
ALTER TABLE videos ADD COLUMN author VARCHAR(50);          -- 作者/发布者名称
ALTER TABLE videos ADD COLUMN description TEXT;            -- 视频简介
ALTER TABLE videos ADD COLUMN category VARCHAR(20);        -- 分类：dogs / cats / training / funny
```

`Video.java` 实体同步添加：
```java
@Column(name = "duration")
private String duration;

@Column(name = "duration_seconds")
private Integer durationSeconds;

@Column(name = "view_count")
private Integer viewCount = 0;

private String author;

@Lob
private String description;

private String category;  // "dogs", "cats", "training", "funny"
```

同时建议提供批量添加视频的管理接口：
```
POST /api/admin/videos
Body: { "title": "...", "url": "...", "productId": 1, "category": "cats", "duration": "03:25", "durationSeconds": 205 }
```

## 12. 商品列表接口返回 shopName（JOIN 优化）

**现状：** `GET /api/products` 返回的商品对象有 `shopId`，但没有 `shopName`。
**问题：** 前端 `ProductCard.vue` 需要显示所属商店名，当前每次都要额外调用 `/shops/{id}` 获取，效率低。

**建议：** 在 `ProductService` 或 `ProductController` 中 JOIN `shops` 表，直接返回 `shopName` 字段：

```json
{
  "id": 1,
  "name": "英短蓝猫",
  "shopId": 1,
  "shopName": "爱宠之家",
  ...
}
```

`Product` 实体添加：
```java
@Transient
private String shopName;  // 非数据库字段，查询时填充
```

在 `listAll()` 或 `findAll()` 查询中 LEFT JOIN shops 表填充此字段。
