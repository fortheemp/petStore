# 后端添加清单 — 第三轮

> 以下均为退单/退款功能需要但后端尚未支持的改进。
> 请在 `petstore-server` 中实现，实现后告知前端同事即可对接。

---

## 1. 退单拒绝时保留拒绝理由

**现状：** `OrderService.approveRefund()` 拒绝退单时执行 `order.setRefundReason(null)`，把用户申请的理由清空了。
**问题：** 前端无法知道退单被拒绝的原因，用户看不到任何拒绝通知。

**修改位置：** `OrderService.approveRefund()` 方法

**改为：**
```java
if (approved) {
    order.setStatus(-3); // 退单成功
    restoreStock(orderId);
} else {
    // 拒绝时保留 refundReason 作为拒绝理由，不清空
    order.setStatus(order.getPreviousStatus());
    order.setPreviousStatus(null);
}
```

即删除 `order.setRefundReason(null);` 这一行，拒绝时保留 `refundReason` 作为拒绝理由。

---

## 2. 退单拒绝 API 支持传递拒绝理由

**现状：** `POST /api/admin/orders/{id}/approve-refund` 只接受 `{ approved: true/false }`，没有理由字段。
**问题：** 管理员拒绝退单时无法填写拒绝理由。

**修改位置：** `AdminController.approveRefund()` 和 `OrderService.approveRefund()`

**Controller 改为：**
```java
@PostMapping("/{id}/approve-refund")
public Result approveRefund(@PathVariable Long id,
                            @RequestBody Map<String, Object> body) {
    boolean approved = (Boolean) body.get("approved");
    String reason = body.getOrDefault("reason", "").toString();
    orderService.approveRefund(id, approved, reason);
    return Result.success(approved ? "退款已处理" : "已拒绝退款");
}
```

**Service 改为：**
```java
public void approveRefund(Long orderId, boolean approved, String reason) {
    Order order = orderRepository.findById(orderId)
            .orElseThrow(() -> new RuntimeException("订单不存在"));
    if (order.getStatus() != -2) {
        throw new RuntimeException("订单状态错误：当前非退单申请中状态");
    }
    if (approved) {
        order.setStatus(-3);
        restoreStock(orderId);
    } else {
        order.setStatus(order.getPreviousStatus());
        order.setPreviousStatus(null);
        // 用管理员填写的理由覆盖用户申请的理由
        if (reason != null && !reason.isEmpty()) {
            order.setRefundReason(reason);
        }
    }
    order.setUpdatedAt(LocalDateTime.now());
    orderRepository.save(order);
}
```

## 实现优先级

| 优先级 | 内容 | 说明 |
|--------|------|------|
| P0 | #1 退单拒绝时保留拒绝理由 | 前端展示拒绝通知的前提 |
| P0 | #2 退单拒绝 API 支持传递拒绝理由 | 管理员需要填写拒绝原因 |

## 前端对接说明

后端实现后，前端需要同步以下改动：
- 管理后台「拒绝」按钮改为弹窗（含理由输入框），调用 `POST /api/admin/orders/{id}/approve-refund` 时传 `{ approved: false, reason: "..." }`
- 前端通过 `status === 2/3 且 refundReason 不为空` 判断退单被拒绝，显示拒绝理由
