# 后端添加清单 — 第二轮

> 以下均为前端当前需要但后端尚未支持的功能。
> 请在 `petstore-server` 中实现，实现后告知前端同事即可对接。

---

## 1. 用户表新增 phone 字段

**现状：** `users` 表没有 `phone` 列，注册时前端传了手机号但后端没存。
**问题：** 无法实现手机号登录，个人资料页也无法显示手机号。

```sql
ALTER TABLE users ADD COLUMN phone VARCHAR(20);
```

`User.java` 实体同步添加：
```java
private String phone;
```

同时建议注册时将手机号存入数据库：
`UserService.register()` 中把 `RegisterDTO.phone` 写入 `user.setPhone()`。

---

## 2. 登录接口支持手机号

**现状：** `POST /api/user/login` 只通过 `username` 字段查询，只能用户名登录。
**需要改为：** 自动识别输入类型，支持用户名或手机号登录。

**实现方式：**
```java
// UserService.login() 修改
public User login(String account, String password) {
    User user = userRepository.findByUsername(account);
    if (user == null) {
        user = userRepository.findByPhone(account);
    }
    if (user == null || !user.getPassword().equals(password)) {
        throw new RuntimeException("用户名或密码错误");
    }
    return user;
}
```

`UserRepository` 新增：
```java
User findByPhone(String phone);
```

前端无需改动，登录表单已兼容。

---

## 3. 评价接口返回用户头像

**现状：** `GET /api/products/{id}/reviews` 返回 `username`，没有 `avatar`。
**问题：** 前端评价区只能显示用户名首字母作为头像，无法展示真实头像。

**建议：** 在 `ReviewService` 或 `ReviewController` 中 JOIN `users` 表，返回 `avatar` 字段：

```json
{
  "id": 1,
  "username": "管理员",
  "avatar": "/uploads/images/xxx.png",
  "rating": 5,
  "content": "很好",
  "createdAt": "2026-06-26T10:30:00"
}
```

---

## 4. 用户资料接口返回 phone 字段

**现状：** `GET /api/user/profile/{id}` 不返回 `phone`。
**问题：** 前端个人资料页手机号显示为空。

实现 #1 后，确保 `UserService.getProfile()` 或 Controller 中将 `phone` 字段包含在响应中即可。

---

## 5. 注册接口保存手机号

**现状：** `RegisterDTO` 有 `phone` 字段，但 `UserService.register()` 没有将它存入数据库。
**问题：** 注册时填了手机号但实际没存，后续无法使用。

实现 #1 后，在 `register()` 方法中加入：
```java
user.setPhone(dto.getPhone());
```

## 实现优先级

| 优先级 | 内容 | 说明 |
|--------|------|------|
| P0 | #1 users 表加 phone 字段 | 其他功能的前置依赖 |
| P0 | #2 登录支持手机号 | 用户需求 |
| P0 | #5 注册保存手机号 | 配合 #1 |
| P1 | #4 个人资料返回 phone | 配合 #1 |
