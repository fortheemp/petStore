# PetStore 部署指南

## 本地开发

### 启动后端

```bash
cd petstore-server
mvn spring-boot:run
```

后端运行在 `http://localhost:8080`

### 启动前端

```bash
cd petstore-frontend
npm install
npm run dev
```

前端运行在 `http://localhost:5173`，API 自动代理到 8080。

---

## 外网部署（内网穿透）

### 前置条件

- 后端已启动（`localhost:8080`）
- 已安装 frp 客户端并配置好隧道

### 第一步：构建前端

```powershell
cd petstore-frontend
$env:VITE_API_BASE_URL="/api"; npm run build
```

### 第二步：复制到后端 static 目录

```powershell
rm -r ../petstore-server/src/main/resources/static/assets
rm ../petstore-server/src/main/resources/static/index.html
cp -r dist/* ../petstore-server/src/main/resources/static/
```

### 第三步：重启后端

重启 Spring Boot，后端会同时托管前端页面和 API。

### 第四步：frp 隧道配置

| 隧道名称 | 远程端口 | 本地端口 | 说明 |
|---------|---------|---------|------|
| petstore_api | 53577 | 8080 | API 接口 |
| petstore_web | 46077 | 8080 | 前端页面 + API |

两个隧道都指向 8080，前端和 API 同源，无跨域问题。

### 第五步：访问

- 前端页面：`https://frp-hat.com:46077`
- API 接口：`https://frp-hat.com:53577`

---

## 注意事项

1. **每次改完前端代码都要重新 build 并复制到 static 目录，重启后端**
2. 手机 APP（uni-app）直接连接后端，不受 CORS 限制，无需额外配置
3. `static/` 目录下的前端文件是部署副本，源代码在 `petstore-frontend/` 目录
4. `.env.development` 用于本地开发（API 走 `/api` 代理）
5. `.env.production` 用于外网访问（API 走 `https://frp-hat.com:53577`）
6. 本地测试外网版本时，用 `VITE_API_BASE_URL=/api` 构建即可

## 快捷启动（本地）

```bash
# 终端1：后端
cd petstore-server && mvn spring-boot:run

# 终端2：前端
cd petstore-frontend && npm run dev
```

或使用 `start.bat` 一键启动（需已安装 Maven 和 Node.js）。
