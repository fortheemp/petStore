# PetStore 部署指南

## 本地开发

### 启动后端

```bash
cd petstore-server
mvn spring-boot:run
```

后端运行在 `http://localhost:8080`

### 启动前端

```bash
cd petstore-frontend
npm run dev
```

前端运行在 `http://localhost:5173`，API 请求通过 Vite 代理自动转发到 8080。

---

## 外网访问（内网穿透）

### 前置条件

- 后端已启动（`localhost:8080`）
- 前端 dev server 已启动（`localhost:5173`）
- 已安装 frp 客户端并配置好隧道

### frp 隧道配置

| 隧道名称 | 远程端口 | 本地端口 | 说明 |
|---------|---------|---------|------|
| petstore_api | 53577 | 8080 | API 接口（移动端用） |
| petstore_web | 46077 | 5173 | 前端页面（浏览器访问） |

### 访问地址

- 前端页面：`https://frp-hat.com:46077`
- API 接口：`https://frp-hat.com:53577`

---

## 备用方案：静态部署

如果不想开 dev server，也可以把前端 build 后放到后端 static 目录：

```powershell
cd petstore-frontend
$env:VITE_API_BASE_URL="/api"; npm run build

rm -r ../petstore-server/src/main/resources/static/assets
rm ../petstore-server/src/main/resources/static/index.html
cp -r dist/* ../petstore-server/src/main/resources/static/

cd ../petstore-server
mvn spring-boot:run
```

此时 frp 两个隧道都指向 8080。

---

## 注意事项

1. **推荐方式**：前端 dev server + frp 指向 5173，改代码自动热更新，无需重新 build
2. `vite.config.js` 已配置 `allowedHosts: ['frp-hat.com']`，允许外网访问
3. `.env.development` 使用 `/api` 作为 API 基址，走 Vite 代理
4. `.env.production` 使用 `https://frp-hat.com:53577` 作为 API 基址，直接连后端
5. 手机 APP（uni-app）直接连接后端 `https://frp-hat.com:53577`，无需额外配置
